## Jila

An experimental game engine I am working on to make my own video games.

## All technologies we have access to in this engine:

1- [Assimp](https://www.assimp.org/):

Open Asset Import Library is a library to load various `3d file formats` into a shared, in-memory intermediate format.
It supports more than 40 file formats for import and a growing selection of file formats for export.

2- [BGFX](https://bkaradzic.github.io/bgfx/):

Cross-platform, graphics API agnostic, “Bring Your Own Engine/Framework” style rendering library. supported rendering
backends:
<ul>
<li>Direct3D 11</li>
<li>Direct3D 12</li>
<li>GNM (only for licensed PS4 developers, search DevNet forums for source)</li>
<li>Metal</li>
<li>OpenGL 2.1</li>
<li>OpenGL 3.1+</li>
<li>OpenGL ES 2</li>
<li>OpenGL ES 3.1</li>
<li>Vulkan</li>
<li>WebGL 1.0</li>
<li>WebGL 2.0</li>
</ul>

3- [CUDA](https://developer.nvidia.com/cuda-zone/):

A parallel computing platform and programming model to run on GPUs.

4- [EGL](https://www.khronos.org/egl):

An interface between Khronos rendering APIs such as OpenGL ES or OpenVG and the underlying native platform window
system. It handles graphics context management, surface/buffer binding, and rendering synchronization and enables
high-performance, accelerated, mixed-mode 2D and 3D rendering using other Khronos APIs.

5- [FMOD](https://www.fmod.com/):

An end-to-end solution for adding sound and music to any game.

6 - [FreeType](https://freetype.org/):

Font rendering engine.

7- [GLFW](https://www.glfw.org/):

Provides a simple API for creating windows, contexts and surfaces, receiving input and events. Handle user input (
keyboard, mouse, gaming peripherals) and manage contexts. Also features multi-monitor support, clipboard access, file
drag-n-drop, and more.

8- [HarfBuzz](https://harfbuzz.github.io/):

A text shaping library that allows programs to convert a sequence of Unicode input into properly formatted and
positioned glyph output — for any writing system and language.

9- [hwloc](https://www.open-mpi.org/projects/hwloc/):

The Portable Hardware Locality (hwloc) software package provides a portable abstraction (across OS, versions,
architectures, ...) of the hierarchical topology of modern architectures, including NUMA memory nodes (DRAM, HBM,
non-volatile memory, CXL, etc.), processor packages, shared caches, cores and simultaneous multithreading. It also
gathers various system attributes such as cache and memory information as well as the locality of I/O devices such as
network interfaces, InfiniBand HCAs or GPUs.

10- [JAWT](https://docs.oracle.com/javase/8/docs/technotes/guides/awt/AWT_Native_Interface.html):

The AWT native interface.

11- [Jemalloc](https://jemalloc.net/):

A general purpose malloc implementation that emphasizes fragmentation avoidance and scalable concurrency support.

12- [KTX (Khronos Texture)](https://www.khronos.org/ktx/):
A lightweight container for textures for OpenGL®, Vulkan® and other GPU APIs.

13- [libdivide](https://libdivide.com/):

A library that replaces expensive integer divides with comparatively cheap multiplication and bit-shifts.

14- [LLVM](https://llvm.org/):

A collection of modular and reusable compiler and toolchain technologies.

15- [LMDB](https://www.symas.com/lmdb):

An extraordinarily fast, memory-efficient database. With memory-mapped files, it has the read performance of a pure
in-memory database while retaining the persistence of standard disk-based databases.

16- [LZ4](https://lz4.org/):

A lossless data compression algorithm that is focused on compression and decompression speed.

17- [meow_hash](https://github.com/cmuratori/meow_hash):

An extremely fast non-cryptographic hash.

18- [meshoptimizer](https://github.com/zeux/meshoptimizer):

A mesh optimization library that makes meshes smaller and faster to render.

19- [msdfgen](https://github.com/Chlumsky/msdfgen):

A multi-channel signed distance field generator. A utility for generating signed distance fields from vector shapes and
font glyphs, which serve as a texture representation that can be used in real-time graphics to efficiently reproduce
said shapes.

20- [NanoVG](https://github.com/memononen/nanovg):

Anti-aliased 2D vector drawing library on top of OpenGL for UI and visualizations.

21- [NFD](https://github.com/LWJGL/lwjgl3/tree/master/modules/lwjgl/nfd):

A small C library that portably invokes native file open, folder select and file save dialogs.

22-[Nuklear](https://github.com/Immediate-Mode-UI/Nuklear) :

A minimal-state, immediate-mode graphical user interface toolkit.

23-[ODBC](https://learn.microsoft.com/en-us/sql/odbc/microsoft-open-database-connectivity-odbc?view=sql-server-ver16):

A C programming language interface that makes it possible for applications to access data from a variety of database
management systems (DBMSs).

24-[OpenAL](https://www.openal.org/):

A cross-platform 3D audio API appropriate for use with gaming applications and many other types of audio applications.

25-[OpenCL](https://www.khronos.org/opencl/):

cross-platform, parallel programming of diverse processors found in personal computers, servers, mobile devices and
embedded platforms.

26-[OpenGL](https://www.khronos.org/opengl/):

The most widely adopted 2D and 3D graphics API in the industry, bringing thousands of applications to a wide variety of
computer platforms. It is window-system and operating-system independent as well as network-transparent.

27-[OpenGLES](https://www.khronos.org/opengles/):

cross-platform API for rendering advanced 2D and 3D graphics on embedded and mobile systems - including consoles,
phones, appliances and vehicles. It consists of a well-defined subset of desktop OpenGL suitable for low-power devices,
and provides a flexible and powerful interface between software and graphics acceleration hardware.

28-[OpenVR](https://github.com/ValveSoftware/openvr):

an API and runtime that allows access to VR hardware from multiple vendors without requiring that applications have
specific knowledge of the hardware they are targeting. This repository is an SDK that contains the API and samples. The
runtime is under SteamVR in Tools on Steam.

29-[OpenXR](https://www.khronos.org/openxr/):

a royalty-free, open standard that provides a common set of APIs for developing XR applications that run across a wide
range of AR and VR devices. This reduces the time and cost required for developers to adapt solutions to individual XR
platforms while also creating a larger market of easily supported applications for device manufacturers that adopt
OpenXR.

30-[OPUS](https://opus-codec.org/):

a totally open, royalty-free, highly versatile audio codec. Opus is unmatched for interactive speech and music
transmission over the Internet, but is also intended for storage and streaming applications. It is standardized by the
Internet Engineering Task Force (IETF) as RFC 6716 which incorporated technology from Skype’s SILK codec and Xiph.Org’s
CELT codec.

31-[LibOVR](https://developer.oculus.com/documentation/) and [OpenVR](https://github.com/ValveSoftware/openvr):

`LibOVR` is the API of the Oculus SDK. And `OpenVR` is an API and runtime that allows access to VR hardware from
multiple vendors without requiring that applications have specific knowledge of the hardware they are targeting.

32-[PAR](https://prideout.net/):

[octasphere](https://prideout.net/blog/octasphere/) is for generating triangle meshes of spheres, rounded boxes, and
capsules. [PAR Shapes](https://prideout.net/shapes) is to generate parametric surfaces and other simple shapes. And
finally, [PAR StreamLines](https://prideout.net/blog/par_streamlines/) is to triangulate wide lines and curves.

33-[Remotery](https://github.com/Celtoys/Remotery):

A realtime CPU/GPU profiler hosted in a single C file with a viewer that runs in a web browser.

34-[RPMalloc](https://github.com/mjansson/rpmalloc):

A public domain cross-platform lock free thread caching 16-byte aligned memory allocator implemented in C.

35-[ShaderC](https://github.com/google/shaderc):

A collection of tools, libraries, and tests for Vulkan shader compilation.

36-[SPIRV-Cross](https://github.com/KhronosGroup/SPIRV-Cross):

A practical tool and library for performing reflection on SPIR-V and disassembling SPIR-V back to high level languages.

37-[SSE](https://www.osgi.org/):

LWJGL OSGi bundle (SSE bindings)

38-[STB](https://github.com/nothings/stb):

It's a collection of single-file public domain libraries written in C/C++. It currently includes these libs:
<ul>
<li>stb_easy_font</li>
Quick-and-dirty easy-to-deploy bitmap font for printing frame rate, etc.
<li>stb_image</li>Image loading/decoding from file/memory: JPG, PNG, TGA, BMP, PSD, GIF, HDR, PIC.
<li>stb_image_resize</li>
Resize images larger/smaller with good quality.
<li>stb_image_write</li>
Image writing to disk: PNG, TGA, BMP
<li>stb_perlin</li>
Revised Perlin noise (3D input, 1D output).
<li>stb_rect_pack</li>
Simple 2D rectangle packer with decent quality.
<li>stb_truetype</li>
Parse, decode, and rasterize characters from truetype fonts.
<li>stb_vorbis</li>
Decode ogg vorbis files from file/memory to float/16-bit signed output.
</ul>

39-[TinyEXR](https://github.com/syoyo/tinyexr):

Tiny OpenEXR image loader/saver library

40-[TinyFD](https://sourceforge.net/projects/tinyfiledialogs/):

Tiny File Dialogs. a native dialog library with support for:

* message / question
* input / password
* save file
* open file & multiple files
* select folder
* color picker

41-[Tootle](https://github.com/GPUOpen-Archive/amd_tootle):

A 3D triangle mesh optimization library that improves on existing mesh preprocessing techniques.

42-[VMA](https://gpuopen.com/vulkan-memory-allocator/):

Vulkan Memory Allocator. The industry-leading, open source, memory allocation library for the Vulkan API.

43-[Vulkan](https://www.vulkan.org/):

A new generation graphics and compute API that provides high-efficiency, cross-platform access to modern GPUs used in a
wide variety of devices from PCs and consoles to mobile phones and embedded platforms.

44-[XXHash](https://github.com/Cyan4973/xxHash):

An Extremely fast Hash algorithm, running at RAM speed limits.

45-[Yoga](https://www.yogalayout.dev/):

An embeddable and performant flexbox layout engine.

46-[ZSTD](https://facebook.github.io/zstd/):

A fast lossless compression algorithm, targeting real-time compression scenarios at zlib-level and better compression
ratios.

47-[JOML](https://github.com/JOML-CI/JOML):

A Java math library for OpenGL rendering calculations

48-[Steam](https://github.com/code-disaster/steamworks4j):

A thin Java wrapper to access the Steamworks API
